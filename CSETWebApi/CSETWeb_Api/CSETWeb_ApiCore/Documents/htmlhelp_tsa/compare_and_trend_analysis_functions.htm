Compare and Trend Analysis Functions

There are two types of aggregation; Trend and Compare.
 
•  Trend shows cumulative changes to a facility over time by analyzing several assessments of the same facility completed, say, annually. The analysis result is a report.
 
•  Compare looks at multiple assessments to determine their strengths and weaknesses, which creates a summary compliance report. This would be helpful for a facility consisting of several completed assessment. The analysis result is a report.
 
Load Assessment Files:
 
To begin, add the assessment files to be analyzed. Click the Select Assessments button. In the Select Assessments popup, select the desired assessment files. At least two files must be present in order for the aggregation function to be performed. It is recommended that no more than five assessments be analyzed at a time. The figure below shows the screen with files
added and the compatibility score.
Figure: Select Assessments popup
 
The compatibility estimates reflect the similarity of the standards addressed by the loaded
assessments. It is not an estimate of how alike the answers are among the assessments. A low compatibility percentage is defined by having little to minimal overlap in questions or
requirements contained within the loaded standards between the assessments.
 
Click "OK" to see the Assessment table.
 
Figure: Assessments table
 
1)   This table lists the assessments to be included in the aggregation. The parts of the table are:
 
Alias: The Alias is a short nickname used to quickly identify the assessment in the analysis charts. The prepopulated name may be changed by clicking within the box and typing the desired alias.
 
Assessment Name: The user-entered name supplied under the Information tab during the original assessment.
 
Assessment Date: The user-entered date supplied under the Information tab during the original assessment.
 
2)   The standards table lists all the standards used by the included assessments and indicates which assessments used which standards. It uses the Alias to identify the assessments.
 
Perform the Analysis
 
Once the assessment file table is complete, click the Next button to begin the analysis.
 
For Trend and Compare:
 
After clicking next the results screen will load. Compare will have multiple sub-headings of information to review. Click Generate Report to create an HTML report of the results which will include the text entered under the aggregation results and the Mode and SAL from the Default assessment.
 
 
Trend
 
Once the assessment file table is complete, click the Next button to begin the Trend analysis.
 
Trend Results: The analysis for a Trend aggregation presents several graphs. The first graph displays overall compliance by percent ranged over the time covered by the loaded assessments, with components and standards broken out separately.
 
Figure: Trend Overall Percent Compliance Score
 
The next graphs show the top five most and least improved security topic areas, again over the time covered by the assessments analyzed.
 
Figure: Trend Most Improved Areas chart
 
Figure: Top Areas of Concern chart
 
 
The next graph breaks out compliance of each assessment by category. The A, B, C, etc., is the alias assigned to an assessment file.
Figure: Category Percent Comparisons chart
 
An HTML report may be created by clicking the Generate Trend Report button. The report will include the information from this analysis screen and the Mode and SAL from the default assessment.
 
 
Compare
 
Once the assessment file table is complete, click the Next button to begin the Compare analysis.
 
Compare Results: The analysis for a Compare aggregation presents several graphs. This
analysis is more complex than that for Trend. The Compare screen has additional tabbed headings. These are Overall Summary, Commonly Missed Questions, Individual Assessment Comparison, and Best to Worst.
 
An HTML report may be created by clicking the Generate Compare Report button. The report will include the information from the analysis screens and the Mode and SAL from the default
assessment.
 
Overall Summary: This tab is opened by default and presents compliance information by
standard versus component and by security category.
 
Figure: Compare Overall Average Summary chart
 
Figure: Compare Standards Answers chart
 
Figure: Compare Components Answers chart
 
Figure: Compare Category Average for All Assessments chart
 
Commonly Missed Questions: This tab presents a table of missed questions from all the assessments analyzed listed by category and subcategory. A missed question is one either answered No. Answers of Yes, N/A, ALT, or Unanswered are not included in the table.
 
Figure: Compare Commonly Missed Questions tab
 
Individual Assessment Comparison: This screen is similar to the Overall Summary, except that it breaks out the information by assessment. A table shows the total answers given by assessment. Next, a chart displays compliance by components and standards, by assessment. The last graph on the left provides the SAL of each assessment to aid in identifying the high-risk facilities.
 
Figure: Compare Results Individual Assessment Comparison table
 
Figure: Compare Overall Comparison chart
 
Figure: Compare Comparison of Security Assurance Levels
 
Figure: Compare Category Percent Comparisons chart
 
Best to Worst: This screen requires the user to select a topic on the left before the graph on the right is populated. The figure below provides an example with Account Management selected.
 
Figure: Compare Best to Worst
 
The graph presents the percent of each answer option by assessment. The bars are ordered to show the most compliant assessment at the top and the least compliant at the bottom. The best is considered to have the largest number of Yes and ALT answers among the included assessments.
 
 
 
 
 
Resource Library
The Resource Library is an excellent way to help the user better understand and resolve the concerns identified by the assessment and to improve the security of the user's systems. It contains a variety of standards, reports, templates, white papers, plans, and other cybersecurity-related documents. The figure below shows the Resource Library window.
Figure: Resource Library window
 
Search Screen
 
Two ways are available to find documents within the Resource Library. This section discusses the Search feature. The other way is by using the document tree structure discussed in the help section titled Browse Screen.
 
The Search screen option of the Resource Library provides a way to find a list of documents based on the text string typed into the search box. Clicking the Search tab opens a search box. Enter the desired text string and click on the magnifying glass icon or press the keyboard Enter key to begin the search.
 
The figure below shows an example where the user has typed in the string "contingency." In this case, CSET TSA searches through all the documents for occurrences of the word "contingency" and then ranks and presents them in an ordered list in the Search Results.
 
Figure: Resource Library Search screen
 
Search tab: Clicking the Search tab will display the search functions of the Resource Library. The Resource Library always opens to the Search tab.
