Cloning a Module

To identify a cloned module in the Module List (Standard and Question Set Builder main page), the original module’s title is copied to the new clone, appending the word “(copy)” to the title.
Clicking on the newly cloned filename will take you to the Module Detail page where you can fine-tune details such as Module Name, Short Name, Description, and industry Category.
 
There are two ways to clone a module in the Module Builder: the Clone button on the Module List (the Standard And Question Set Builder) page, and the Clone From Existing Module(s) button on the Module Detail page.
Clone Module Button
 
To clone an existing module, begin by clicking the Clone Module Button for the module you wish to clone. A copy of the module is created including requirements and questions which can be tailored to your specific industry or business function.
 
The second option for cloning an existing module can be found on the Module Detail page.
To clone a module, click the Clone from Existing Module(s) button on the lower right of the Module Detail page.
Figure: Module Detail
After starting the cloning function, a popup menu allows you to select from a list of possible modules for cloning.
Figure: Select Module Screen
 
After choosing a module to clone (or more than one, if applicable) select the Add Questions
and Requirements button in the popup. This will integrate your chosen assessment materials into your cloned assessment.
 
After cloning your chosen assessment material, you can further modify your cloned assessment.
 
From the Module Detail page and the Requirements, Questions, and Manage Documents
buttons at the bottom right hand of the screen, you can begin editing and customizing your
module using the Editing steps below.
 
 
Editing the content of a cloned module
 
After completing the cloning step and editing the Module Details for your cloned module, you can edit the content of the module including the Requirements Details, Security Assurance Levels, Supplemental Information, References, and Related Questions.

Deleting a Custom Module
 
Delete Module Button
 
A custom module may be deleted, provided no other modules have requirements based on it. This restriction is enforced to maintain CSET TSA assessment data integrity.
 
 
 
 
Add Requirements
Clicking the Requirements button opens the Requirement Listing screen where the user can add a new requirement to their set.
 
Figure: Requirement Listing screen
 
Select the Create Requirement button to open the Add requirement dialogue. All fields are required to add a new requirement.
 
Figure: Add a Requirement dialogue
 
Category Dropdown list: Select or enter the control group category. Typing in this field will filter the dropdown values to matching values.
 
For more information about question categories, see the Assessment Categories help section.
 
Question Group Heading Dropdown list: This is a value under which questions are grouped when displayed in CSET TSA questions mode.
 
For more information about question group headings, see the Assessment Categories help section.
 
Subcategory Dropdown list: Select or enter the control subcategory.
 
For more information about question subcategories, see the Assessment Categories help section.
 
Title/Identifier field: Enter the title or identifier of the Requirement. For example, Requirement 1 (Req.1).
 
Requirement Text field: Enter the full text of the Requirement. Line breaks are preserved for readability and are presented when answering in CSET TSA Requirements mode.
 
Create button: Click the Create button to save [PIJ2] the new requirement and jump to the Requirement Detail screen.
 
Requirement Detail[PIJ3] 
Figure: Requirement Details and SAL level
 
The values for Identifier/Title and the Requirement text are editable on this screen as well.
 
Security Assurance Level: Select all Security Assurance Levels that are applicable to the Requirement.
 
For more information about Security Assurance Levels, see the Security Assurance Level (SAL) help section.
 
Next, add any supplemental information for the requirement.
Figure: Supplemental text box
 
Supplemental Information text box: Enter any supplemental information for the Requirement in this box. The text can be formatted using the controls and can be edited directly as HTML by clicking the </> button.
 
References
Figure: Add and manage documents
 
Manage Documents button: Before a reference document can be connected to a Requirement it must first be associated with the Module. Click this button to jump to the Standard Documents List, where that connection can be made.
 
Source Documents dropdown list: A Source Document is the primary location that supplies the Requirement for the Standard. Documents that have been associated with this Module will be listed here.
 
Bookmark: Optional. CSET TSA will render hyperlinks in the References section of the Requirements screen. For convenience, those links will open the Reference or Help document and jump directly to a pre-defined bookmark. If there is a bookmark in the document, it can be entered here. Enter the bookmark without a leading hash/pound symbol (#).
 
Click the ‘+’ button to add the reference to the Requirement. Multiple references can be added to a Requirement.
 
Help Documents dropdown list: A Help Document is a secondary source of information that may be helpful to the assessor to help understand a Requirement. They are added the same way as Source Documents and are listed separately on the Requirements screen in CSET TSA.
 
At this point, the user can add simple questions to the individual requirement, so that they can utilize both Questions and Requirements mode. (This is not required)
 
To learn more about Questions/Requirements mode, see the Mode Selection help section.
